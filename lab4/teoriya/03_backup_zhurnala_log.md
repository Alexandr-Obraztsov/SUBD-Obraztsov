# Резервная копия журнала транзакций (LOG backup)

## Теория

Резервная копия журнала (`BACKUP LOG`) копирует записи журнала транзакций с момента последнего лог-бэкапа (или с момента полного бэкапа, если лог ещё не бэкапили). Доступна только при модели восстановления **FULL** или **BULK_LOGGED**. Непрерывная цепочка лог-бэкапов позволяет восстанавливать базу к произвольному моменту времени (point-in-time restore).

Параметры аналогичны полному бэкапу: `TO DISK`, `INIT`, `COMPRESSION`, `STATS`. Файлы обычно имеют расширение `.trn`.

---

## Как это сделано в лабораторной

После перевода ProjectDB_Lab4 в FULL и создания полного бэкапа выполняется первый лог-бэкап:

```sql
BACKUP LOG [ProjectDB_Lab4]
TO DISK = N'/var/opt/mssql/backup/ProjectDB_Lab4_log_1.trn'
WITH
    INIT,
    COMPRESSION,
    STATS = 10;
GO
```

В цепочке восстановления порядок такой: полный бэкап → (при необходимости) лог-бэкапы → дифференциальные. В данной лабораторной для восстановления на втором экземпляре используется цепочка full → diff1 → diff2 → diff3; лог-бэкап демонстрирует возможность при необходимости дополнить цепочку для восстановления к моменту времени.
